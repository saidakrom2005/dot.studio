=== PRICE CALCULATION FORMULA ===
Base Price Calculation:
1. Start with base price per m²: $120
2. Multiply by room type factor (0.7 - 1.0)
3. Multiply by style factor (1.0 - 1.4)
4. Add wall type cost ($0 - $100)
5. Add floor type cost ($0 - $150)

Example:
20 m² x $120 x 0.8 (living room) x 1.2 (modern) + $50 (wallpaper) + $100 (wood floor) = $3,480

=== CUSTOMIZING PRICES ===
Edit the priceMultipliers object in script.js:

    this.priceMultipliers = {
        room: {
            kitchen: 1.0,          // Adjust multipliers here
            'living-room': 0.8,
            bedroom: 0.7,
            bathroom: 0.9
        },
        style: {
            minimalistic: 1.0,      // Adjust multipliers here
            modern: 1.2,
            classic: 1.4,
            scandinavian: 1.3,
            industrial: 1.1
        },
        wallType: {
            painted: 0,             // Adjust costs here
            wallpaper: 50,
            brick: 100,
            concrete: 80
        },
        floorType: {
            laminate: 0,            // Adjust costs here
            wood: 150,
            tile: 120,
            concrete: 100
        }
    };

    this.basePrice = 120;           // Adjust base price per m²

=== CONNECTING TO YOUR CRM ===
1. In script.js, find the sendToCRM() method around line 220
2. Replace 'https://your-crm-api.com/api/requests' with your CRM endpoint
3. Add authentication headers if needed:

    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_API_TOKEN',
        'X-API-Key': 'YOUR_API_KEY'
    }

4. Test with your CRM's API documentation

=== IMAGE LAYERING SYSTEM ===
The preview uses 5 CSS layers that update in real-time:

1. layer-base        → Wall texture pattern
2. layer-walls       → Main wall color
3. layer-flooring    → Floor type pattern
4. layer-furniture   → Style indicator icon
5. layer-decor       → Additional decorative elements

No external images needed - all patterns generated with CSS gradients!
To add actual images: Replace the CSS backgrounds with image URLs:
    wallsLayer.style.backgroundImage = "url('img/wall-texture.jpg')";

=== ADDING NEW ROOM TYPES ===
1. Add button in HTML:
    <button class="toggle-btn" data-room="office">Office</button>

2. Add to priceMultipliers in script.js:
    office: 1.1

3. The system auto-detects it!

=== ADDING NEW STYLES ===
1. Add option in HTML:
    <option value="bohemian">Bohemian</option>

2. Add to priceMultipliers in script.js:
    bohemian: 1.25

3. Add icon to updateLayersPreview():
    bohemian: '◈'

=== LOCAL STORAGE (SAVED DESIGNS) ===
Designs are saved to browser's localStorage as JSON.
To view saved designs in console:
    JSON.parse(localStorage.getItem('designs'))

To clear all saved designs:
    localStorage.clear()

=== DATA PERSISTENCE ===
Current implementation uses localStorage (browser-based).
For server persistence, modify saveDesign() to send data to your backend:

    fetch('/api/save-design', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify(design)
    })

=== COMMON ISSUES & SOLUTIONS ===

Issue: Price updates aren't showing
Solution: Check console for errors (F12 > Console)

Issue: CRM request fails
Solution: Verify CORS headers in your CRM API
Add: Access-Control-Allow-Origin: *

Issue: Carousel items won't scroll
Solution: Check for overflow-x: hidden on parent

Issue: Images not showing on mobile
Solution: Use CSS patterns instead of images
or serve mobile-optimized versions

Contact support if issues persist!
